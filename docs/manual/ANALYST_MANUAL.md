## 4. 核心功能模块

### 4.1 实时期权链

**读取方式：**
1. 左侧输入框输入标的代码（如 `AAPL`、`SPY`）
2. 按 `Enter` 加载期权链
3. 使用 `←/→` 切换到期月

**颜色编码：**
- 🟢 绿色单元格 = IV相对低（适合买方）
- 🔴 红色单元格 = IV相对高（适合卖方）
- 🟡 黄色行 = 成交量异常（VOI > 2x）
- 🔵 蓝色行 = 大额保费流入（>$100K）

### 4.2 IV曲面分析

**正常曲面特征：**
- Put Skew: 低行权价IV > ATM IV（正常的保险需求）
- Term Structure: 近期IV < 远期IV（正向期限结构）

**异常信号：**
- 曲面倒置 = 近期事件驱动（财报/FDA/FOMC）
- 极端Skew = 市场对冲需求激增
- IV曲面"飞翼"结构 = 尾部风险定价

## 5. Greeks解读指南

### 5.1 一阶Greeks（日常使用最频繁）

**Delta (Δ) — 价格敏感度**
- Call Delta范围: 0 到 +1
- Put Delta范围: -1 到 0
- |Delta| > 0.70 = 深度ITM，高度跟随标的
- |Delta| ≈ 0.50 = ATM，最大时间价值
- |Delta| < 0.20 = 深度OTM，高杠杆低概率

**实战用法：**
```
建仓Delta中性组合:
  例：持有1手Call（Delta=+0.45）
    需卖出 0.45/1 = 0.45手标的（或对应期货）对冲
    使净Delta≈0
```

**Theta (Θ) — 时间衰减（每日）**
- 期权卖方的"租金收益"
- ATM期权 Theta 最大（衰减最快）
- 规律：Theta 在最后30天加速衰减
- 经验法则：30DTE以内，每天损失约3× vs 60DTE

**Vega (ν) — 波动率敏感度（每1% IV变化）**
- 买期权者：Vega > 0（IV上升获益）
- 卖期权者：Vega < 0（IV下降获益）
- IV Crush（财报后IV骤降）是卖方的最大朋友

### 5.2 二阶Greeks（专业分析必备）

**Gamma (Γ) — Delta变化率**
- ATM + 短期 = Gamma最大（最"爆炸性"）
- 做市商持有正Gamma（买期权）→ 卖高买低对冲
- 做市商持有负Gamma（卖期权）→ 追涨杀跌对冲

```
Gamma暴露计算:
GEX = Gamma × OI × 100 × S²
正GEX区间: 做市商平抑波动
负GEX区间: 做市商放大波动 → Gamma挤压触发区
```

**Vanna (∂²V/∂S∂σ) — Delta关于IV的敏感度**
- IV上升时，Call Delta增加，Put Delta减少（绝对值增加）
- 应用：VIX飙升时，Vanna驱动大量做市商Delta对冲流量
- 月度VIX期货到期日前后：Vanna流量最大

**Charm (∂Δ/∂T) — Delta时间衰减**
- OTM期权随时间接近到期：Delta向0漂移
- 应用：周五（到期日）做市商Charm流量 → 股价漂向大型OI集中行权价

### 5.3 三阶Greeks（进阶策略）

**Speed (∂Γ/∂S)** — Gamma变化率
- 预判Gamma敏感度随股价移动的变化
- 用于管理Gamma scalp的风险

**Zomma (∂Γ/∂σ)** — Gamma关于IV的敏感度
- IV变化对Gamma对冲需求的影响
- VIX飙升时特别重要

**Color (∂Γ/∂T)** — Gamma时间衰减
- 接近到期时Gamma如何快速衰减

## 6. Smart Money信号判读

### 6.1 信号分类说明

| 类型 | 特征 | 交易意义 |
|------|------|----------|
| **投机单** (🔴) | 扫单 + OTM + 短DTE + 高VOI | 方向性押注，可跟随方向 |
| **套保单** (🔵) | 买Put + 长DTE + ATM + 大保费 | 机构对冲，不代表看跌 |
| **做市单** (⚫) | 小尺寸 + 双边 + ATM + 超短DTE | 做市商流量，忽略 |
| **价差单** (🟣) | 同时Buy+Sell不同行权价 | 需看整体结构 |

### 6.2 鲸鱼单判读流程

```
收到 >$1M 大单信号
    │
    ├─ 是否为扫单 (Sweep)?
    │     ├─ 是 → 急迫方向性押注 → 高度关注
    │     └─ 否 → 可能套保或价差腿 → 需综合判断
    │
    ├─ 行权价位置?
    │     ├─ OTM > 5% → 高杠杆押注，事件驱动
    │     ├─ ATM ± 2% → 短期方向押注
    │     └─ ITM → 可能合成仓位替代股票
    │
    ├─ 到期时间 (DTE)?
    │     ├─ 1-7天 → 极度投机，事件前夕
    │     ├─ 7-30天 → 短期方向押注
    │     └─ > 60天 → 套保或长期看法
    │
    └─ 是否高于Ask成交?
          ├─ 是 → 买方非常激进，信号更强
          └─ 否 → 正常成交，信号一般
```

### 6.3 净保费流向解读

| PCR（Put/Call Ratio）| 解读 |
|---------------------|------|
| < 0.5 | 极度看多，可能过热 |
| 0.5 - 0.8 | 偏多 |
| 0.8 - 1.2 | 中性 |
| 1.2 - 2.0 | 偏空 |
| > 2.0 | 极度恐慌/对冲 |

## 7. Gamma挤压实战指南

### 7.1 什么是Gamma挤压

当大量散户买入OTM期权时：
1. 做市商被迫卖出期权（持有**负Gamma**）
2. 股价上涨 → 做市商需买入标的对冲Delta
3. 买入标的 → 推高股价 → 触发更多Delta对冲
4. 正反馈循环 → Gamma挤压！

### 7.2 终端预警级别

| 级别 | 条件 | 操作建议 |
|------|------|----------|
| ✅ NORMAL | 正Gamma区间 | 正常交易 |
| 👁 WATCH | 负Gamma区间 | 提高警惕，缩小仓位 |
| ⚠️ WARNING | 距翻转点 < 2% | 减仓，考虑保护性对冲 |
| 🚨 CRITICAL | 距翻转点 < 0.5% | 紧急降低风险敞口 |

### 7.3 翻转点交易策略

```
翻转点 = GEX从正变负的股价水平

多头策略（GEX > 0，稳定区）:
  - 卖出Straddle / Strangle（利用低波动率）
  - 网格期权策略

空头策略（GEX < 0，放大区）:
  - 买入期权（利用波动率放大）
  - 突破翻转点后做趋势

关键价位：
  - Call Wall = 强支撑（做市商在此大量Delta对冲形成买压）
  - Put Wall  = 强压力（做市商在此大量对冲）
```

## 10. 常见问题排查

### Q1: 数据不更新 / 显示N/A

检查步骤：
1. `F5` 强制刷新
2. 检查 `设置 > 数据源 > 连接状态`（应显示绿色）
3. 查看日志：`docker logs options-backend --tail 50`
4. 验证API Key余额：Polygon.io后台查询

### Q2: 3D IV曲面卡顿

- 降低采样密度：`设置 > 可视化 > IV曲面精度` → 改为"标准"
- 关闭实时更新：改为手动刷新（每分钟）
- 检查GPU：确认Chrome使用独立GPU（`chrome://gpu`）

### Q3: Smart Money信号延迟

- Polygon.io免费版有15秒延迟 → 升级至`Starter`套餐（实时）
- 检查WebSocket连接：`设置 > 数据流 > 延迟监控`

### Q4: 回测速度慢

- 启用C++引擎：确认 `options_core.so` 已编译
- 减少期权链密度：限制行权价数量 < 30
- 使用合成期权链（当真实历史链不可用时自动启用）

### Q5: IV显示异常（>200%或<1%）

数据清洗规则已启用，异常IV会自动标红。原因可能是：
- 极深度ITM/OTM期权（流动性差，买卖价差极宽）
- 数据源错误（0.01手成交触发的噪声IV）
- 解决方案：`设置 > 数据质量 > 最小流动性过滤` → 启用
